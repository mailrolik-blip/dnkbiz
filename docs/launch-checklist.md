# Launch Checklist

Чек-лист для первого закрытого запуска на `stage.dnkbiz.ru`.

## Перед деплоем

- Проверьте, что в `main` только нужные изменения для закрытого запуска.
- Убедитесь, что staging не требует `seed` для запуска. `seed` не входит в обычный deploy.
- Подготовьте минимум два аккаунта:
  - обычный пользователь для покупки и проверки доступа;
  - администратор с ролью `ADMIN` для `/admin`.
- Убедитесь, что у платного курса есть:
  - опубликованная страница курса;
  - активный тариф;
  - при необходимости открытые ознакомительные уроки.

## После деплоя

- Главная страница открывается на `https://stage.dnkbiz.ru/`.
- `GET https://stage.dnkbiz.ru/api/health` отвечает `{"status":"ok"}`.
- Нет явных ошибок 500 в браузере и серверных логах.

## Базовый пользовательский маршрут

- Регистрация нового пользователя работает.
- Вход под новым пользователем работает.
- Каталог открывается и фильтры работают.
- Страница платного курса открывается.
- Ознакомительные уроки платного курса открываются.
- Бесплатный курс открывается сразу после входа.
- Экран оплаты по СБП открывается.
- На экране оплаты видны:
  - QR-код СБП;
  - сумма;
  - номер заказа;
  - пояснение, что доступ откроется после ручной проверки.
- Кнопка `Я оплатил` переводит заказ в статус ручной проверки.
- В личном кабинете заказ виден как ожидающий проверки.

## Проверка ручной оплаты

- `/admin` открывается только у пользователя с ролью `ADMIN`.
- Обычный пользователь не может попасть в `/admin` и уходит в `/lk`.
- В `/admin` заказ появляется в очереди `PROCESSING`.
- `PROCESSING` понимается как "деньги еще не подтверждены, доступ не открыт".
- Кнопка подтверждения оплаты переводит заказ в `PAID`.
- После подтверждения:
  - заказ исчезает из очереди ручной проверки;
  - в блоке `Доступы` появляется доступ к нужному курсу;
  - пользователь видит открытый курс.
- Кнопка отклонения перевода переводит заказ в `FAILED`.
- После отклонения доступ к курсу не открывается.

## Контент и админка

- В `/admin/help` открывается инструкция для администратора.
- Редактирование названия и описания курса работает.
- Редактирование урока работает.
- Изменение порядка уроков работает.
- Изменение признака ознакомительного урока работает.
- Изменение цены и активности тарифа работает.
- После изменения курса публичная страница курса открывается без ошибок.

## Публичные тексты

- На публичных страницах используется `Бизнес школа ДНК` или `ДНК`.
- В пользовательском UI нет:
  - `DNK Biz`;
  - `DNK`;
  - `LMS`;
  - `staging`;
  - `seed`;
  - `checkout`;
  - `flow`.
- Тексты оплаты честно объясняют:
  - оплата идет по QR СБП;
  - доступ открывается после ручной проверки;
  - проверка может занять время;
  - при вопросах нужно писать в контакты.

## Mobile Smoke Check

- Главная страница читаема на мобильном экране.
- Каталог читаем и карточки не ломают сетку.
- Страница курса читаема и CTA-кнопки доступны.
- Личный кабинет читаем и основные кнопки видны без горизонтального скролла.
- Экран оплаты не ломается на мобильном, QR и кнопки остаются доступны.

## Локальные команды перед коммитом

- `npm run lint`
- `npx tsc --noEmit`
- `npm run build`

Если `npm run build` падает из-за локального `EPERM` в `.next`:

- остановите активный `next dev`;
- удалите `.next`;
- повторите `npm run build`.
